# UI Component Library

Design system built with **shadcn/ui + Tailwind CSS v4**, documented in Figma and implemented in Next.js.  
Stories verify visual parity, interaction states, and accessibility in a single place.

---

## Quick start

```bash
npm run storybook      # dev server → localhost:6006
npm run build-storybook  # export static HTML (ส่งให้ทีม QA ได้โดยไม่ต้องลง Node)
```

## สำหรับทีม Dev

### เพิ่ม story ใหม่

```tsx
// stories/MyComponent.stories.tsx
import type { Meta, StoryObj } from "@storybook/react"
import { MyComponent } from "@/components/ui/my-component"

const meta = {
  title: "Components/MyComponent",
  component: MyComponent,
  tags: ["autodocs"],
} satisfies Meta<typeof MyComponent>

export default meta
type Story = StoryObj<typeof meta>

export const Default: Story = {
  args: { label: "Click me" },
}
```

### Path alias

`@/` ชี้ไปที่ root ของโปรเจกต์ — ใช้ได้ใน stories เหมือนใน Next.js app

```tsx
import { Button } from "@/components/ui/button"   // ✅
import { Button } from "../components/ui/button"   // ✅ (ก็ได้)
```

### Component ใหม่จาก shadcn

```bash
npx shadcn@latest add card   # ติดตั้ง component
# แล้วสร้าง stories/Card.stories.tsx ตาม pattern เดิม
```

## สำหรับทีม QA

### Checklist ต่อ component

เปิด story แต่ละอัน แล้วตรวจตาม checklist นี้:

**1. Visual — ⚡ Playground**
- [ ] ใช้ Controls panel เปลี่ยน `variant` / `size` / `disabled` ครบทุก option
- [ ] toggle **Dark mode** — token ต้องพลิกทันที ไม่มีสีขาดหาย

**2. States — "All States" story**
- [ ] Default state แสดงถูกต้อง
- [ ] Disabled — opacity 50%, คลิกไม่ได้
- [ ] Loading (Button) — spinner หมุน, คลิกไม่ได้
- [ ] Error / aria-invalid (Input) — border เปลี่ยนเป็น destructive

**3. Accessibility — แท็บ A11y**
- [ ] เปิดแท็บ **Accessibility** ด้านล่าง → ต้องไม่มี Violations
- [ ] ทุก interactive element มี focus ring เมื่อ Tab
- [ ] Icon-only buttons ต้องมี aria-label

**4. Tokens — Tokens / Semantic Colors**
- [ ] เปิด story **Component Tokens** → toggle Light/Dark → swatch ต้องพลิกทั้งหมด
- [ ] ไม่มีสี hardcode (`#FFFFFF`, `bg-white`) ปนอยู่ใน component
